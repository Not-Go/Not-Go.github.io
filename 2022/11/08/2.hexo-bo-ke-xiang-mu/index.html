Hexo博客项目

Hexo博客项目环境搭建

首先下载node.js、Git

Node.js下载地址：https://nodejs.org/en/download/

Git下载地址：https://git-scm.com/

注：git的使用请自行百度，这里不教

使用下面命令进行hexo框架和配置Git工具的下载

npm install hexo-cli -g

初始化Hexo项目文件，然后我们在你创建的地址下就可以看到新的文件夹Myblog

hexo init Myblog //Myblog为项目的文件夹名

本地测试项目是否可以运行

hexo clean //清空已经存在的hexo网站文件
hexo generate(可简写g) //依据网页文件和新的css样式生成新的网站文件
hexo server(可简写为s) //启动本地服务器，可以在localhost：4000查看网站修改效果
hexo deploy(可简写为d) //自动生成网页静态文件，并部署指定仓库

补充：我们首先要cd到博客目录，然后才能执行hexo指令，否则都会提示没有该命令

当我们用上图命令后，得出了静态观看网站，就可以ctrl+鼠标点击进去看我们网站的效果了

自此hexo博客的基本骨架就建成了

使用下面的命令查看npm、node、hexo的版本号，以免后续的接口和架构等级不支持

npm -v
node -v
hexo -v

如果hexo的版本低于5.0的话，使用下面的命令进行升级，因为后续的主题有些插件需要5.0的hexo框架

npm install -g hexo-cli(重安装会更新为最新版本)

在Gitee或者Github上创建项目，这里使用gitee(国内访问快，但是敏感词很严，后续换github,这里只是演示)

[请参考官方文档]：将 Hexo 部署到 GitLab Pages | Hexo

然后我们重新部署hexo服务，然后去gitee开启gitee pages服务

步骤简言之：

  1. 开辟仓库
  2. 创建HTTPS或SSH连接(HTTPS支持gitee，SSH支持Github)
  3. 复制HTTPS或者SSH放入Hexo配置文件config里
  4. 修改type、repo、branch的指
  5. 调用Hexo命令，尝试部署

然后通过我们开启Gitee Pages服务的到的网站地址，就可以发给别人看你的网站了

这里提一下，我们后续会进行Hexo骨架美化，我们可以用hexo s命令，把博客改好了再进行更新pages，否则老更新很费神

新建分类categories页

categories 页是用来展示所有分类的页面，也就是导航上的分类页面，如果在你的博客 source 目录下还没有 categories/index.md 文件，那么你就需要手动或者使用命令新建一个，命令如下：

hexo new page "categories"

编辑你刚刚新建的页面文件 /source/categories/index.md，至少需要以下内容：

---
title: categories
date: 2022-11-08 12:30:30
type: "categories"
layout: "categories"
---

新建分类tag页

tags 页是用来展示所有标签的页面，如果在你的博客 source 目录下还没有 tags/index.md 文件，那么你就需要新建一个，命令如下：

hexo new page "tags"

编辑你刚刚新建的页面文件 /source/tags/index.md，至少需要以下内容：

---
title: tags
date: 2022-11-08 12:30:30
type: "tags"
layout: "tags"
---

中文链接转拼音插件

如果你的文章名称是中文的，那么 Hexo 默认生成的永久链接也会有中文，这样不利于 SEO，且 gitment 评论对中文链接也不支持。我们可以用 hexo-permalink-pinyin Hexo 插件使在生成文章时生成中文拼音的永久链接。安装命令如下：

npm install hexo-permalink-pinyin --save

在 Hexo 根目录下的 _config.yml 文件中，新增以下的配置项：

permalink_pinyin:
  enable: true
  separator: '-' # default: '-'

订阅功能

本主题中还使用到了 hexo-generator-feed 的 Hexo 插件来做 RSS，安装命令如下：

npm install hexo-generator-feed --save

在 Hexo 根目录下的 _config.yml 文件中，新增以下的配置项：

feed:
  type: atom
  path: atom.xml
  limit: 20
  hub:
  content:
  content_limit: 140
  content_limit_delim: ' '
  order_by: -date

Pjax功能

Pjax用来使页面不刷新跳转，音乐可以不间断播放。关闭则跳转页面时候刷新页面

pjax:
  on: true
  animation: circle # false, nprogress, circle
  animationColor: 'orangered' # animation为 nprogress时候的动画颜色, red or #000 ...
  cacheBust: false # url 地址追加时间戳，用以避免浏览器缓存
  timeout: 5000    # The timeout in milliseconds for the XHR requests. Set to 0 to disable the timeout.

首页翻转图配置

主题yml文件中修改swiper为以下：

swiper:
  on: true # 设置为false, 则首页不显示轮播图

首页轮播图显示默认内容：

#这里只给部分内容，具体看我下面给的图和theme文件的注释
swiper:
  on: true # 设置为false, 则首页不显示轮播图
  defaultImg: 'https://pic3.zhimg.com/80/v2-7e6795a0b7b4fbe87b0a388b8a052f78_1440w.jpg' # 图片，没有则随机选取
  defaultVideo: '' # video，min-width:992px show
  defaultTile: '' # 标题
  defaultDesc: [''] #描述，例如 ['天行健'，'君子以自强不息']
  defaultLeftBtnText: '阅读文档' # 左边按钮文字, 空字符串则不显示
  defaultRightBtnText: '我的简历' # 右边按钮的文字，空字符串则不显示

defaultImg代表默认图片，defaultVideo代表默认视频，这两个值都设置的话，则在pc端显示视频，手机端显示图片

音乐功能配置

music:
  on: true
  server: tencent   #require music platform: netease, tencent, kugou, xiami, baidu
  type: song    #require song, playlist, album, search, artist
  id: 0020dKPG3mp0GT     #require song id / playlist id / album id / search keyword

server代表音乐平台，netease代表网易云, tencent代表qq音乐
type代表歌曲类别，song代表单曲, playlist代表歌单
id代表歌曲或者歌单的id，例如网易云的歌单地址https://music.163.com/#/playlist?id=2788431035, 后面的id就是这个

这里重点说几个参数，theme是主题色参数, bgFloor是首页背景颜色或者背景图片参数, bgImg是文章详情页面的背景图片或者背景颜色参数, contentOpacity是文章详情页面的左侧文章内容背景颜色的透明度，tocOpacity是文章详情页面的右侧文章目录背景颜色的透明度，其余参数上面有注释哈

特别问题解决：

错误可能一：生成rsa丢到github的ssh里，如果还错就删除rsa重新生成

git config --unset --global user.name
git config --unset --global user.email
git config --unset --global user.autocrlf
git config --global user.name "github用户名"
git config --global user.email "github邮箱"
ssh-keygen -t rsa -C "github邮箱"  #一直回车
ssh -T git@github.com #测试效果
